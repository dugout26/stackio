## 1. 사업 개요

### 1.1 사업명

**STACK.io** — 브라우저 기반 멀티플레이어 서바이벌 .io 게임

### 1.2 사업 비전

> "글로벌 .io 게임 시장에서 한국 개발 IP로 경쟁하는, 설치 없이 즉시 플레이 가능한 차세대 캐주얼 멀티플레이어 게임 플랫폼"

### 1.3 핵심 가치 제안

| 구분 | 내용 |
|------|------|
| **즉시 접근성** | 다운로드/설치 없이 브라우저(PC + 모바일)에서 URL 클릭만으로 즉시 플레이 |
| **전략적 깊이** | Vampire Survivors(2022 Steam 올해의 게임) 자동 공격 메커닉 + .io 게임 PvP를 결합한 독창적 게임 루프 |
| **공정한 수익 모델** | 모든 유료 아이템은 순수 코스메틱 — Pay-to-Win(과금 우위) 완전 배제 |

### 1.4 사업 단계

**MVP(최소 기능 제품) 개발 100% 완료** — 베타 서비스 런칭 준비 단계

### 1.5 핵심 지표 요약

| 항목 | 현황 |
|------|------|
| 게임 무기 | 기본 6종 + 진화 6종 = **총 12종** |
| 패시브 업그레이드 | **6종** (Speed, Magnet, Armor, Regen, Critical, Area) |
| 코스메틱 스킨 | 6개 카테고리, **총 34종** |

| 동시 접속 | 방(Room)당 최대 **50명** |
| 서버 성능 | **60 tick/sec** 시뮬레이션, **20 tick/sec** 네트워크 전송 |
| 최적화 | 델타 압축, 공간 해싱, 뷰포트 컬링 |

## 2. 문제 인식 및 시장 기회

### 2.1 시장이 직면한 문제

#### 문제 1: 기존 게임의 높은 진입 장벽

기존 PC/모바일 게임은 **다운로드 → 설치 → 회원가입 → 튜토리얼**까지 평균 5~10분 이상 소요되며, 이 과정에서 잠재 유저의 **60~70%가 이탈**합니다. 반면 브라우저 게임은 URL 클릭 한 번으로 즉시 플레이가 가능하여 진입 장벽이 사실상 제로입니다.

#### 문제 2: .io 게임 시장의 혁신 부재

agar.io(2015), slither.io(2016) 이후 **.io 게임 카테고리에 혁신적인 신작이 부재**합니다. 기존 .io 게임들은 단순한 메커닉(먹기, 피하기)에 머물러 있으며, 2022년 Steam 올해의 게임을 수상한 Vampire Survivors가 입증한 **"자동 공격 + 빌드 전략"** 장르를 멀티플레이어 브라우저 환경에 적용한 사례가 전무합니다.

#### 문제 3: 글로벌 .io 시장에 한국 IP 부재

글로벌 .io 게임 시장 상위권을 미국, 브라질, 유럽 개발사가 독점하고 있으며, **한국 개발사의 .io 게임은 전무**합니다. 한국은 세계 4위 게임 시장이자 e-스포츠 강국으로, 기술력과 콘텐츠 역량을 보유하고 있으나 해당 시장에 진출하지 못하고 있습니다.

### 2.2 왜 지금인가? — 시장 기회의 타이밍

| 요인 | 내용 |
|------|------|
| **기술 성숙** | HTML5 Canvas, WebSocket, Web Audio API 등 브라우저 기술이 성숙하여 60fps 실시간 멀티플레이어 구현이 가능 |
| **인프라 보급** | 5G/고속 인터넷 글로벌 보급으로 200ms 이하 레이턴시에서 전 세계 동시 플레이 가능 |
| **장르 검증** | Vampire Survivors가 1,000만 카피 이상 판매하며 "자동 공격 서바이벌" 장르의 시장 수요를 입증 |
| **웹 게임 재부상** | WebGPU, WebAssembly 등 차세대 웹 기술로 브라우저 게임의 성능 한계가 급격히 해소되는 추세 |
| **SEO 이점** | 브라우저 게임은 검색 엔진 최적화(SEO)와 소셜 공유가 용이하여 자연 유입 경로 확보에 유리 |

## 3. 제품 소개

### 3.1 게임 플레이 핵심 루프

```
① 접속 (브라우저 열기 → 이름 입력 → PLAY)
        ↓
② 이동 (마우스/터치로 방향 지정, 무기 자동 발사)
        ↓
③ 전투 (몹 처치 → XP 획득 → 레벨업)
        ↓
④ 성장 (레벨업 시 무기/패시브 3개 중 1개 선택 → 빌드 전략)
        ↓
⑤ 경쟁 (PvP 전투 + 실시간 리더보드)
        ↓
⑥ 순환 (사망 → 통계 확인 → 재시작/공유)
```

**핵심 재미 요소**: 매 게임마다 무기 조합과 패시브 선택이 달라져 **동일한 게임이 두 번 없는** 높은 리플레이 가치를 제공합니다.

### 3.2 무기 시스템 (총 12종)

| **Bullet** | 가장 가까운 적을 자동 조준 발사 | 15 | 발사 간격: 500ms, 레벨당 +1발 |

이 원칙은 다음과 같은 전략적 이유에 기반합니다:
- 확률형 아이템 규제(게임산업법) 해당 없음
- 신규 유저의 진입 장벽 제거
- 커뮤니티 신뢰 확보 및 장기 리텐션 향상
- 글로벌 시장 진출 시 각국 규제 리스크 최소화

### 6.2 권위적 서버 모델 (Authoritative Server)

STACK.io의 치트 방지 핵심은 **서버가 모든 게임 로직의 유일한 진실 소스(Single Source of Truth)**인 구조입니다.

| 항목 | 처리 주체 | 설명 |
|------|-----------|------|
| 데미지 계산 | 서버 | 클라이언트는 데미지 수치를 전송하지 않음 |
| 충돌 감지 | 서버 | 공간 해싱 기반 O(n) 충돌 판정 |
| XP/레벨 관리 | 서버 | 클라이언트는 선택 인덱스(0/1/2)만 전송 |
| 위치 검증 | 서버 | 클라이언트는 이동 방향(angle)만 전송 |
| HP/사망 판정 | 서버 | 서버에서 계산 후 결과만 클라이언트에 통보 |

**클라이언트가 전송하는 데이터**: `{ angle: number, moving: boolean }` — 이 두 값만으로 전체 게임이 동작하며, 클라이언트의 어떤 데이터도 게임 로직에 신뢰하지 않습니다.

### 6.3 성능 최적화 (구현 완료)

| 최적화 기법 | 설명 | 효과 |
|------------|------|------|
| **델타 압축** | 변경된 엔티티만 전송, 5초마다 전체 스냅샷 | 네트워크 트래픽 70~80% 절감 |
| **짧은 속성명** | `x, y, a, h, l, s` 등 1~2글자 | 패킷 크기 50% 절감 |
| **공간 해싱** | 100px 그리드 기반 충돌 감지 | O(n²) → O(n) 성능 향상 |
| **뷰포트 컬링** | 화면 밖 엔티티 렌더링 제외 | GPU 부하 60~70% 절감 |
| **모바일 감지** | `IS_MOBILE` 플래그로 이펙트 감소 | 저사양 기기 60fps 유지 |
| **오브젝트 풀링** | 파티클/투사체 재사용 | GC 압력 감소, 프레임 드롭 방지 |

### 6.4 기술 스택 상세

| 계층 | 기술 | 버전 | 선택 이유 |
|------|------|------|-----------|
| **프론트엔드** | HTML5 Canvas + Vanilla JS | ES6+ | 프레임워크 없이 번들 크기 최소화, 로딩 속도 최적화 |
| **백엔드** | Node.js + Express | 18+ / 4.x | 비동기 I/O로 높은 동시접속 처리 |
| **실시간 통신** | Socket.io | 4.7 | WebSocket + 폴백, 자동 재연결 |
| **계정 DB** | SQLite (sql.js) | 1.10 | 서버리스, 외부 DB 서비스 불필요 |
| **리더보드** | Redis | 4.6 | Sorted Set으로 O(log N) 순위 연산 |
| **결제** | Stripe | 14.14 | 글로벌 결제, Checkout 세션 + 웹훅 |
| **인증** | 자체 구현 | - | crypto.scrypt 해싱, UUID 세션, httpOnly 쿠키 |

> **의존성 최소화**: package.json 기준 프로덕션 의존성 **단 7개**. 경량 아키텍처로 유지보수 부담과 보안 취약점을 최소화했습니다.

### 6.5 봇 AI 시스템

실제 플레이어가 부족할 때 자연스러운 게임 경험을 보장하기 위한 지능형 봇 시스템:

| 행동 상태 | 조건 | 행동 |
|-----------|------|------|
| **Roam** (탐색) | 기본 상태 | 맵을 탐색하며 이동 |
| **Farm** (파밍) | 근처에 몹 존재 | 몹에게 접근하여 경험치 획득 |
| **Hunt** (사냥) | HP 높고 + 약한 적 발견 | 다른 플레이어 공격 |
| **Flee** (도주) | HP 낮음 | 위협으로부터 도주 |
| **Collect** (수집) | 근처에 XP 구슬 | XP 구슬 수집 |

- **3가지 성격 유형**: Aggressive(공격적), Farmer(파밍형), Balanced(균형형)
- **스마트 레벨업**: 진화 우선 → 새 무기(3개 미만) → 무기 강화 → 패시브
- **최소 10인 유지**: 실제 플레이어 부족 시 봇 자동 충원
